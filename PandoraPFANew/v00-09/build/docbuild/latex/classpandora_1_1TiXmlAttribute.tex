\subsection{Detailed Description}
An attribute is a name-value pair. 

Elements have an arbitrary number of attributes, each with a unique name.

\begin{Desc}
\item[Note:]The attributes are not Ti\-Xml\-Nodes, since they are not part of the tiny\-XML document object model. There are other suggested ways to look at this problem. \end{Desc}

\subsubsection{\setlength{\rightskip}{0pt plus 5cm}int pandora::Ti\-Xml\-Attribute::Query\-Int\-Value (int $\ast$ {\em \_\-value}) const}\label{classpandora_1_1TiXmlAttribute_25f321f86a68edb64a377994a9cdfe5e}


Query\-Int\-Value examines the value string. 

It is an alternative to the \doxyref{Int\-Value()}{p.}{classpandora_1_1TiXmlAttribute_b22a5e9b9b6b50deaa73cbc7b0450344} method with richer error checking. If the value is an integer, it is stored in 'value' and the call returns TIXML\_\-SUCCESS. If it is not an integer, it returns TIXML\_\-WRONG\_\-TYPE.

A specialized but useful call. Note that for success it returns 0, which is the opposite of almost all other Tiny\-Xml calls.
