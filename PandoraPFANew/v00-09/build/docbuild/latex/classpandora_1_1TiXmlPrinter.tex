\subsection{Detailed Description}
Print to memory functionality. 

The \doxyref{Ti\-Xml\-Printer}{p.}{classpandora_1_1TiXmlPrinter} is useful when you need to:

\begin{enumerate}
\item Print to memory (especially in non-STL mode)\item Control formatting (line endings, etc.)\end{enumerate}


When constructed, the \doxyref{Ti\-Xml\-Printer}{p.}{classpandora_1_1TiXmlPrinter} is in its default \char`\"{}pretty printing\char`\"{} mode. Before calling Accept() you can call methods to control the printing of the XML document. After \doxyref{Ti\-Xml\-Node::Accept()}{p.}{classpandora_1_1TiXmlNode_aaef4731878c5ebabd50fc9675b8f0d7} is called, the printed document can be accessed via the \doxyref{CStr()}{p.}{classpandora_1_1TiXmlPrinter_e35d9790667fbc8bc2d1df3cfbac63b4}, \doxyref{Str()}{p.}{classpandora_1_1TiXmlPrinter_cd8ec8ca87c6e3fc260ae52a74b9b423}, and \doxyref{Size()}{p.}{classpandora_1_1TiXmlPrinter_704630bc73e72e69890544730b75fc38} methods.

\doxyref{Ti\-Xml\-Printer}{p.}{classpandora_1_1TiXmlPrinter} uses the Visitor API. 

\footnotesize\begin{verbatim}	TiXmlPrinter printer;
	printer.SetIndent( "\t" );

	doc.Accept( &printer );
	fprintf( stdout, "%s", printer.CStr() );
	\end{verbatim}
\normalsize
