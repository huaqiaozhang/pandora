\subsection{Detailed Description}
Implements the interface to the \char`\"{}Visitor pattern\char`\"{} (see the Accept() method. 

) If you call the Accept() method, it requires being passed a \doxyref{Ti\-Xml\-Visitor}{p.}{classpandora_1_1TiXmlVisitor} class to handle callbacks. For nodes that contain other nodes (Document, Element) you will get called with a Visit\-Enter/Visit\-Exit pair. Nodes that are always leaves are simply called with \doxyref{Visit()}{p.}{classpandora_1_1TiXmlVisitor_c79e5e0cc60dc3f7d1447c4c2c3b772f}.

If you return 'true' from a Visit method, recursive parsing will continue. If you return false, {\bf no children of this node or its sibilings} will be Visited.

All flavors of Visit methods have a default implementation that returns 'true' (continue visiting). You need to only override methods that are interesting to you.

Generally Accept() is called on the \doxyref{Ti\-Xml\-Document}{p.}{classpandora_1_1TiXmlDocument}, although all nodes suppert Visiting.

You should never change the document from a callback.
